# Лабораторная работа №3 | Вариант 33
## Стеганография: скрытие текста в изображении

### Описание
Программа реализует три режима работы с методом LSB-стеганографии:

1. **Декодирование готового изображения** — извлечение текста из файла преподавателя (`new33.png` + `keys33.txt`)
2. **Кодирование своего текста** — встраивание сообщения в изображение методом варианта 33
3. **Декодирование своего файла** — извлечение текста из созданного в пункте 2 изображения

### Требования к исходному изображению
- Формат: PNG (без потерь, иначе младшие биты будут повреждены)
- Цветовая модель: RGB
- Размер: достаточный для вмещения сообщения (2 пикселя на символ)

### Установка зависимостей
```bash
pip install pillow
```

### Использование
1. Запустите программу:
```bash
python3 3lab.py
```

2. Выберите нужный пункт меню (1 / 2 / 3 / q)

3. Введите имена файлов и текст сообщения (для пункта 2)

4. Программа покажет результат и вернётся в меню

### Особенности варианта 33
- **Метод**: `b3-B, b2-B, b1-B, b0-B` — биты 3, 2, 1, 0 байта синего канала
- **Кодирование**: в младшие 4 бита канала B записывается ниббл (полбайта) сообщения
- **Расход пикселей**: 2 пикселя на один символ (старший и младший ниббл)
- **Визуальные изменения**: минимальны, старшие биты B сохраняются

### Схема кодирования символа
```
Символ 'H' = 0x48 = 01001000

  Старший ниббл: 0100  →  в b3..b0 канала B пикселя №1
  Младший ниббл: 1000  →  в b3..b0 канала B пикселя №2

Пиксель 1:  B = 10101011  →  B = 10100100
Пиксель 2:  B = 10011011  →  B = 10011000
```

### Особенности программы
- **Обработка ошибок**: программа не падает при некорректных файлах, битых координатах, вводе вне cp1251
- **Простота**: следует стилю примеров из лекций преподавателя
- **Graceful exit**: корректная обработка Ctrl+C и Ctrl+D
- **Защита исходных данных**: результаты пункта 2 сохраняются в отдельные файлы (`new33_my.png`, `keys33_my.txt`), исходные файлы преподавателя не затираются
- **Цикл меню**: после каждой операции программа возвращается к выбору режима

### Структура кода
- Чтение и парсинг файла ключей
- Безопасное открытие изображений с конвертацией в RGB
- Декодирование по полному байту B (пункт 1, метод из лекции)
- Кодирование по методу варианта 33 (пункт 2)
- Декодирование по методу варианта 33 (пункт 3)
- Многоуровневая обработка исключений

### Доказательство корректности
При кодировании (пункт 2) для первого символа сообщения программа выводит:
- код символа и его биты
- значения старшего и младшего ниббла
- координаты двух пикселей-ключей
- исходные значения B до изменения
- новые значения B после изменения

### Результат выполнения
Программа выводит:
- Результат декодирования (пункт 1 и 3): скрытый текст
- При кодировании (пункт 2): доказательство корректности для первого символа и имена созданных файлов
- Сообщения об ошибках с понятным русским текстом при некорректном вводе
