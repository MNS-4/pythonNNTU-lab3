# Лабораторная работа №3. Стеганография

Учебный проект по курсу «Программирование на Python».
Реализация кодирования и декодирования текстовых сообщений, скрытых в изображениях методом LSB-стеганографии.

**Вариант: 33**

---

## Задание

Составить программу на Python, которая реализует:

1. Декодирование текста из закодированного изображения, предоставленного преподавателем (`new33.png` + `keys33.txt`).
2. Кодирование своего текста в изображение по методу своего варианта.
3. Декодирование своего закодированного изображения.

Для доказательства корректности кодирования при работе программа выводит:

- биты первого символа текстового сообщения;
- исходные значения пикселей;
- изменённые значения пикселей.

---

## Метод кодирования (вариант 33)

Согласно таблице вариантов: **`b3-B, b2-B, b1-B, b0-B`** — биты 3, 2, 1, 0 байта, отвечающего за синий цвет (канал B в модели RGB).

То есть в младшие 4 бита канала B каждого пикселя-ключа записывается половина байта (ниббл) сообщения:

- **первый пиксель пары** хранит старший ниббл символа (биты 7..4);
- **второй пиксель пары** хранит младший ниббл символа (биты 3..0).

Для одного символа требуется **2 пикселя-ключа**.

### Схема кодирования

```
Символ 'H' = 0x48 = 01001000

  Старший ниббл: 0100 (4)  →  записывается в b3..b0 канала B пикселя №1
  Младший ниббл: 1000 (8)  →  записывается в b3..b0 канала B пикселя №2

Пиксель 1:  B = 10101011  →  B = 10100100   (младшие 4 бита заменены на 0100)
Пиксель 2:  B = 10011011  →  B = 10011000   (младшие 4 бита заменены на 1000)
```

Старшие 4 бита канала B сохраняются, поэтому визуальные искажения минимальны.

## Запуск

```bash
python3 3lab.py
```

Появится меню:

```
Выберите режим стеганографии:
1. Декодировать текст из готового изображения
2. Закодировать свой текст в изображение
3. Декодировать свой текст из изображения
q. Выход
```

### Пункт 1 — декодирование файла преподавателя

Запрашивает имена `new33.png` и `keys33.txt`, выводит сообщение, скрытое преподавателем.

Метод: **полный байт B** каждого пикселя-ключа интерпретируется как ASCII-код символа (метод из презентации лекции).

### Пункт 2 — кодирование своего сообщения

Запрашивает имя исходной картинки и текст сообщения. Создаёт:

- `new33_my.png` — картинку со скрытым сообщением;
- `keys33_my.txt` — файл с координатами изменённых пикселей.

Метод: **4 младших бита** канала B, 2 пикселя на символ (по варианту 33).

### Пункт 3 — декодирование своего файла

Запрашивает `new33_my.png` и `keys33_my.txt`, достаёт сообщение обратным преобразованием по варианту 33.

## Защита от падений

Программа устойчива к некорректному вводу и нештатным ситуациям:

- несуществующие файлы изображений и файлов ключей;
- файлы, которые не являются изображениями;
- битые строки в файле ключей (мусор, пустые строки, не числа);
- координаты пикселей вне изображения (включая отрицательные);
- пустое сообщение при кодировании;
- символы сообщения, не входящие в cp1251 (автоматическое переключение на utf-8);
- сообщение, не помещающееся в картинку;
- отсутствие прав на запись выходных файлов;
- прерывание пользователем (Ctrl+C, Ctrl+D);
- нечётное количество ключей при декодировании.

После любой ошибки программа возвращается в меню и готова к следующей операции. Выход — только по выбору пункта `q`.
